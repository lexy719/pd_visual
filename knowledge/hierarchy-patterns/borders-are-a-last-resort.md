## borders-are-a-last-resort
tags: hierarchy, borders, spacing, color, layout

Separate with space, then with background, then with a shadow. Reach for a border last.

**Problem it solves** — borders are the default way to say "these things are different", and they are the noisiest. A page of boxed sections reads as an admin panel regardless of its palette.
**When to use a border at all** — when space and background have failed: a genuine input affordance, a table, or where a hairline carries real meaning.
**Structure** — Try in order: whitespace between groups → a subtle background shift on one group → a soft shadow to lift it → a border only if none of those separated it.
**Why it works** — every border adds a line the eye must process and a rectangle that competes with the content inside it. Space separates just as clearly and costs nothing visually — which is why expensive-looking layouts have so few lines.
**Expected visual outcome** — the page reads as open and considered rather than compartmentalised; groups are still obviously distinct, but the page has far fewer visible edges.
**Examples** — a feature grid separated by generous gaps and a card background instead of outlined boxes.
