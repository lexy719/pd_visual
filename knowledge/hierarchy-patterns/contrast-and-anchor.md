## contrast-and-anchor
tags: hierarchy, clarity, focal-point, scanning, typography, layout

One dominant focal point, one or two secondary anchors. The eye should never have to choose.

**Problem it solves** — pages with too many equal-weight elements read as noisy and indecisive. Hierarchy is what turns a set of claims into a page.
**When to use it** — the page must communicate priorities quickly, or several claims compete for the same attention.
**Structure** — Primary claim → Supporting proof → Secondary details.
**Why it works** — a page is understood faster when it has one visual centre and a few supporting cues. Rank is read before content is.
**Expected visual outcome** — the page reads as calm, confident and easy to scan; one element is unmistakably first.
**Examples** — a bold hero statement supported by a lower-contrast feature row and a small proof band.
